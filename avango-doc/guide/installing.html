Installing Avango Development Environment

For Avango developers working under Windows, it is recommended to compile Avango locally with Microsoft Visual Studio.

This can be done in the following steps.

1. Installing prerequisites
The following software modules have to be downloaded, installed and configured prior to Avango compilation:
  • Microsoft Visual Studio 2005 or 2008 (also known as VS 8 and VS 9)
  • Python programming language 2.5 or later
  • scons build system 1.0.0 or later
  • boost C++ library 1.35 or later. It is recommended to select all variants all the library for Visual Studio 2005
  • OpenSceneGraph 3D graphic toolkit 2.6.0 or later. It is also recommended to compile the toolkit locally. To do that one should download and install Cmake (version 2.4.6 or later), download the prebuilt dependencies, include their bin directory to the system's PATH variable and configure and compile OpenSceneGraph following the instructions of the README.txt in its root directory.

2. Configuring paths
Include the locations of Python, Python/scripts, OpenSceneGraph/bin to the system's PATH environment variable. Include the locations of OpenSceneGraph/lib and OpenSceneGrpah/include to the Visual Studio library and include directories.

3. Downloading and Compiling Avango
Avango source code can be downloaded using an SVN client (such as TortoiseSVN) from the following address: http://avango.org/svn/trunk.

You have the following possibilities of compiling Avango:
  • Open the supplied solution file avango.sln and build the project in the Visual Studio 2008
  • Create a new project from existing code in the Visual Studio:
    • Select the menu "File -> New -> Project from Existing Code"
    • Select the directory in which you sources for avango for Windows reside.
    • Enter the project name
    • Select "Use external build system"
    • As build command, enter the scons command
    • Build the solution
  • Build Avango modules from the Visual Studio command prompt:
    • Open the Visual Studio Command prompt
    • Go to the location where the top level SConstruct file resides.
    • Enter the scons command (you can also build only selected modules by providing the module name after the scons command, e.g. scons core, scons osg, scons python, scons unittest, etc)
In case the compilation was successfull, you should be able to start the SnowLand example (it is located in TBA).

In order to be able to use the built dlls (e.g. to "import avango" into a python application), the so-called manifest file has to be included into the dll. The command for an executable is
mt.exe -manifest MyApp.exe.manifest -outputresource:MyApp.exe;1
for a dll it is
mt.exe -manifest MyLibrary.dll.manifest -outputresource:MyLibrary.dll;2
