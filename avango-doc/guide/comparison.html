Avango and other VR-frameworks

The tables below describe the most popular frameworks for the development of Virtual Reality applications (as found on the web-sites of their respected producers) and shows their relationships with Avango, when needed.

Complete frameworks
Virtual Environments department of Fraunhofer IAIS Avango AVANGO is a free, cross-platform software framework designed for interactive, distributed Virtual Reality applications. It supports a large range of setups, ranging from desktop PCs to large-scale immersive VR installations. AVANGO facilitates rapid development of immersive 3D applications by offering efficient techniques for many common VR tasks. The key features of Avango that differentiate it from other frameworks are: easy creation and control of dynamic 3D scenes, scripting support, built-in multimodal interaction techniques, support for numerous interaction devices and display setups, possibilities to develop distributed VR applications. Avango is licensed under the LGPL.
Virtual Reality Applications Center of the Iowa State University VR Juggler VR Juggler is freely available open source, community-oriented virtual reality application development framework. It is scalable from simple desktop systems like PCs to complex multi-screen systems running on clusters or on high-end workstations and supercomputers. The flexibility of VR Juggler allows applications to execute in many VR system configurations including desktop VR, HMD, CAVE-like devices, and Powerwall-like devices. The portability of the VR Juggler code ensures that any operating system can be used. VR Juggler 2.0 supports IRIX, Linux, Windows, FreeBSD, Solaris, and Mac OS X. It is released under the GNU LGPL.

Although VR Juggler support render distribution via NetJuggler, in contrast to AVANGO it has no direct high-level support for application distribution.
Illinois Simulator Laboratory of University of Illinois at Urbana-Champaign (2 main developers) Syzygy Syzygy is a programming toolkit for writing virtual reality or other graphical applications to run on clusters of networked computers. The rendering of your virtual world is synchronized on all of the computers (to within a few milliseconds), so different screens in the cluster give you different simultaneous views on the virtual world. Syzygy runs on Windows, Linux, MacOS X, and Irix. Syzygy applications can be written in either of two programming languages: C++ or Python.
4 individual developers Ogre3D OGRE (Object-Oriented Graphics Rendering Engine) is a scene-oriented, flexible 3D engine written in C++ designed to make it easier and more intuitive for developers to produce applications utilising hardware-accelerated 3D graphics. The class library abstracts all the details of using the underlying system libraries like Direct3D and OpenGL and provides an interface based on world objects and other intuitive classes. OGRE is design-led rather than feature-led. Every feature that goes into OGRE is considered throughly and slotted into the overall design as elegantly as possible and is always fully documented. It is made available under the LGPL License and works under Windows, Linux and Mac OSX.
Team of developers (~20 persons) affiliated with the Visualization and Animation Group of Virginia Tech University DIVERSE toolkit (DTK) DIVERSE is a cross-platform, open source, API for developing virtual reality applications that can run almost anywhere. It currently runs on Linux, IRIX, Windows XP, and Mac OS X. The goal of DIVERSE is to enable developers to quickly build applications that will run on the desktop as well as various immersive systems. To avoid hindering development DIVERSE is designed without the "center of the universe" paradigm; you only use the parts of DIVERSE that you need. DIVERSE provides: a common user interface to interactive graphics and/or VE programs, a common API to VE oriented hardware such as trackers, wands, joysticks, and motion bases, many other tools such as "remote shared memory", message passing, and collaboration tools., basic "cluster" support through DADS. DIVERSE includes user applications to load, view, and navigate models in the CAVE as easily as the desktop.
Team of developers (~20 persons) of INRIA MOAIS group and the Fundamental Informatics Laboratory of the University or Orleans FlowVR The goal of the FlowVR library is to provide users with the necessary tools to develop and run high performance interactive applications on PC clusters and Grids. The main target applications include virtual reality and scientific visualization. FlowVR enforces a modular programming that leverages software engineering issues while enabling high performance executions on distribued and parallel architectures. The FlowVR software suite has 3 main components: FlowVR: the core middleware library, FlowVR Render: a parallel rendering library, VTK FlowVR: a VTK / FlowVR / FlowVR Render coupling library that enables to render VTK applications using FlowVR Render.
Bunraku/Siames team of IRISA, laboratory of INRIA OpenMask The OpenMask platform is a state of the art technology and will allow you to create powerful simulations thanks to VR device and OpenMask Platform. OpenMASK (Modular Animation and Simulation Kit) is a platform for modular applications development and execution in animation, simulation and virtual reality fields. The main features provided by OpenMASK concern its capabilities: to interact with a virtual environment using various devices (sensors, data gloves, haptic devices) in a local or distant 3D cooperation (collaborative work); to integrate different computation modules (mechanical simulators, human motion, behavioural models) using high level abstractions; to automatically distribute computation on a network of workstations; to display results on an immersive environment (Reality Center, Workbench) or on a standard display. It uses Performer, OpenSG and Ogre for 3D visualization, MAX/MSP and OpenAL for sound generation, and VRPN for integration with VR devices. It supports Windows and Linux and is available under the QPL license.

Panda3D Panda3D is a game engine which includes graphics, audio, I/O, collision detection, and other abilities relevant to the creation of 3D games. Panda includes the following subsystems: particle systems, physics and collisions, soft- and hard-skin character animation, 2D and 3D nonuniform rational B-spline support, 2D graphical user interface support, multipass rendering, and real-time shader framework. Scene graphs
>300 idividual contributors OpenSceneGraph The OpenSceneGraph is an OpenSource, cross-platform graphics toolkit for the development of high-performance graphics applications such as flight simulators, games, virtual reality and scientific visualization. It is based around the concept of a SceneGraph, providing an object-oriented framework on top of OpenGL. This frees the developer from implementing and optimizing low-level graphics calls and provides many additional utilities for rapid development of graphics applications. The code is freely available, modifiable and redistributed under the terms of the The OpenSceneGraph Public License (OSGPL), which is based on the LGPL.

AVANGO is based on OpenSceneGraph; it extends the functionality of OpenSceneGraph with its dynamic and distributed scenes, scripting support and the support for different display systems and interaction devices
currently led by 2 individual developers; originated from the Fraunhofer IGD OpenSG OpenSG is a portable scenegraph system to create realtime graphics programs, e.g. for virtual reality applications. It is developed following Open Source principles (LGPL) and can be used freely. It runs on Windows, Linux, Solaris and MacOS X and is based on OpenGL. The features that differentiate OpenSG from other scenegraphs are extensibility, multi-thread safety and clustering capabilities. The detailed comparison with OpenSceneGraph is available here.

Peripheral support
Department of Computer Science of the University of North Carolina at Chapel Hill VRPN: Virtual Reality Peripheral Network The Virtual-Reality Peripheral Network (VRPN) is a set of classes within a library and a set of servers that are designed to implement a network-transparent interface between application programs and the set of physical devices (tracker, etc.) used in a virtual-reality (VR) system. The idea is to have a PC or other host at each VR station that controls the peripherals (tracker, button device, haptic device, analog inputs, sound, etc). VRPN provides connections between the application and all of the devices using the appropriate class-of-service for each type of device sharing this link. The application remains unaware of the network topology. VRPN also provides an abstraction layer that makes all devices of the same base class look the same. At the same time, it is possible for an application that requires access to specialized features of a certain device to derive a class that communicates with this type of device. The VRPN client (application-side) library has been tested on SGI/Irix, PC/Win32, PC/Cygwin, HP700/Hpux, Sparc/Solaris, PC/Linux, Mac/OSX, Ipaq/Linux, and Zaurus/Linux. The server-side library compiles under SGI/Irix, PC/Win32, and PC/Linux. VRPN support is provided in many VR frameworks. The software can be use without license fees; cmmercial support is available through 3rdtech.

AVANGO currently has its own interfaces to physical devices. However, VRPN support could be provided at a later time.

Mechdyne CAVELib CAVELib is a powerful application programmer interface (API) that provides the cornerstone for creating robust interactive three-dimensional environments. The CAVELib API provides the building blocks to handle the intricacies of creating robust applications for Virtual Environments. Some of the items the CAVELib abstracts away for a developer are, window and viewport creation, viewer-centered perspective calculations, displaying to multiple graphics channels, multi-processing and multi-threading, cluster synchronization and data sharing, and stereoscopic viewing. The CAVELib dramatically extends a developer's abilities to create solutions with scalable, immersive 3D representations without having to contend with operating system, display system, and device-level programming details. Furthermore, the CAVELib's networking capabilities allows collaboration, interaction and data sharing with users across the hall or around the world. The CAVELib API is platform independent enabling developers to create high-end virtual reality applications on IRIX, Solaris, Windows, and Linux operating systems. CAVELib-based applications are externally configurable at run-time making an application executable independent of the display system. Applications created with the CAVELib have the ability to run in nearly all immersive display systems.
The High Performance Computing Center of the University of Stuttgart COVISE COVISE stands for COllaborative VIsualization and Simulation Environment. It is an extendable distributed software environment to integrate simulations, postprocessing and visualization functionalities in a seamless manner. From the beginning COVISE was designed for collaborative working allowing engineers and scientists to spread on a network infrastructure. In COVISE an application is divided into several processing steps, which are represented by COVISE modules. These modules, being implemented as separate processes, can be arbitrarily spread across different heterogeneous machine platforms. Major emphasis was put on the usage of high performance infrastructures such as parallel and vector computers and fast networks. COVISE Rendering modules support Virtual environments ranging form workbenches over powerwalls, curved screens up to full domes or CAVEs. The users can thus analyze their datasets intuitively in a fully immersive environment through state of the art visualization techniques including Volume rendering and fast sphere rendering. Physical prototypes or experiments can be included into the analysis process through Augmented Reality techniques. COVISE is widely used in research and industrial projects.

Collaborative Virtual and Augmented Environments department of Fraunhofer FIT MORGAN The MORGAN AR/VR Framework is platform-independent, distributed, and component-based, supporting developers in realizing Mixed Reality prototypes and applications. MORGAN is assembled from a set of individual components including a render engine, external scene graph support, and access to manifold input and output devices. We further developed a light-weight version of MORGAN for handheld devices, such as PDAs and Smartphones, sharing most of the functionality and offering full runtime compatibility with MORGAN (see Figure 2). Therefore, MORGAN Light and MORGAN cannot only process the same scene formats, but may also be part of the same distributed application. Applications developed for MORGAN require no or only minimal effort to be adapted for MORGAN Light. The render engine consists of the framebuffer abstraction layer for OpenGL and Direct3D and the internal scene graph. MORGAN modules also includes Marvin, a stereoscopic AR/VR viewer, and DEVAL, to support all input and output devices. Researchers and developers of AR/VR projects may license the Morgan AR/VR Framework for evaluation, development and distribution. Academic institutes and universities may apply for free licenses for teaching.

ZGDV Avalon Avalon is an open environment for virtual and augmented reality applications. It was designed and built to serve as a platform for various projects and applications, both built in-house and by our project partners. The development process started at the end of 1997 and is still ongoing. One of the main reasons to start the project was the ever present need for a faster and more efficient application development cycle. Therefore the goal was to define abstractions for interaction and behaviour descriptions, which work well for VR and AR applications. Like most traditional toolkits, Avalon uses a scene-graph to organize the data, as well as spatial and logical relations. In addition, a VR application needs to deal with specific behaviour of objects, and the user interaction via assorted input devices. The major drawback of traditional VR-toolkits is the fact that the application developer has to implement a controller to handle all aspects of both behaviour and interaction for most applications anew. To overcome this drawback, we extended the scene-graph concept of a node to a more generic component, which not just holds the data but defines a state, state-changes, and input/output slots for every node. Therefore application development is done by instantiating and connecting scene components, which are provided by the system. It uses OpenSG as rendering backend. In contrast to AVANGO, it does not provide a sophisticated application-level network distribution support.

Avalon, OpenSG and some other frameworks have been integrated into the instantreality framework, a high-performance Mixed-Reality (MR) system, which combines various components to provide a single and consistent interface for AR/VR developers. The binary modules of the framework are available here.
